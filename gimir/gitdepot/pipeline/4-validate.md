# Phase 4 — Validate the implementation against the design

Read `gitdepot/pipeline/00-context.md`, `gitdepot/DESIGN.md`, and the current
code.

For each part of the design, state whether the code now realizes it —
IMPLEMENTED, PARTIAL, or MISSING — with `file:symbol` evidence. Be skeptical:
check that what is claimed done is actually reachable by the real code path, not
only by a test fixture. Write the result to `gitdepot/VALIDATION.md` and return a
short summary of what conforms and what still does not.
